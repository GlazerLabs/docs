This guide will help you set up the Thryl API on your local development environment.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** (v16 or higher)
- **npm** or **yarn**
- **PostgreSQL** (v12 or higher)
- **Redis** (v6 or higher)

## Quick Start

### 1. Clone the Repository

```bash
git clone <repository-url>
cd thrylBackend
```

### 2. Install Dependencies

```bash
npm install
```

### 4. Database Setup

Create the PostgreSQL database:

```sql
CREATE DATABASE thryl_db;
```

The application will automatically create the required tables on startup.

### 5. Start the Application

#### Development Mode
```bash
npm run dev
```

#### Production Mode
```bash
npm start
```

The API will be available at `http://localhost:3000`

## Docker Setup

If you prefer using Docker:

### 1. Build the Image

```bash
docker build -t thryl-backend .
```

### 2. Run with Docker Compose

```bash
docker-compose up -d
```

This will start the API along with PostgreSQL and Redis services.

## Verification

To verify that the installation was successful:

1. Check if the server is running: `http://localhost:3000/health`
2. You should see a JSON response indicating the service is healthy

## Troubleshooting

### Common Issues

**Port Already in Use**
If port 3000 is already in use, you can change it by setting the `PORT` environment variable:

```bash
PORT=3001 npm start
```

**Database Connection Issues**
- Ensure PostgreSQL is running
- Verify your database credentials in the `.env` file
- Check if the database exists

**Redis Connection Issues**
- Ensure Redis is running
- Verify Redis connection settings in the `.env` file
